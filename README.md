# lukk-echo-interviewer

一个由困困设计的材料内化访谈 skill。

你读完一本书，看完一篇文章，听完一期播客或视频之后，可能会有一种很熟悉的感觉：

好像懂了，又不确定自己是不是真的懂了。

让你点头，你能点头。让你复述，你会开始借作者的话。让你讲给别人听，脑子里有很多点，但不知道从哪里进入。更麻烦的是，如果你想把它写成一篇文章、讲成一段口播，刚一提笔，就发现自己没有线。

这时候你缺的往往不是又一份总结。

你缺的是一个人来采访你。

`lukk-echo-interviewer` 就是这个采访者。它围绕你给出的材料，一问一答地逼近你的理解：哪些话已经变成你的语言，哪些还停在作者那里；哪些例子真的照亮了材料，哪些只是把话题带走了。

## 核心前提

这个 skill 必须围绕一份具体材料工作。

材料可以是：

- 一本书或其中一章。
- 一篇文章、论文、长帖。
- 一期播客或视频的转录稿。
- 一份课程笔记、读书笔记、摘录。
- 你自己整理出来的一段学习材料。

如果没有材料，只是想泛泛聊人生、职业、账号、写作方向，它不适合。

## 适合谁

适合这些情况：

- 你学完一份材料，想校正自己到底有没有理解。
- 你能说出几个关键词，但还不能用自己的话把原观点讲出来。
- 你想知道自己是在复述作者，还是已经带上了自己的判断。
- 你想把材料里的观点和自己的经验连起来，但不想被个人经历带跑。
- 你想把学到的东西分享出去，却不知道从材料里的哪个点切入。
- 你想写文章、口播稿或分享提纲，但需要先把自己的理解讲出来。

它不适合这些情况：

- 你只想让 AI 总结材料。
- 你想让 AI 直接代写读后感。
- 你没有材料，只想让 AI 陪你泛聊。
- 你需要心理咨询、法律、医疗、金融等专业建议。

## 它怎么工作

它默认一问一答。

不会一开始丢给你一整套题库，也不会先替你把材料讲完。它通常会从一个低门槛问题开始：

```text
这份材料里，最卡住你、最亮、或最让你想反驳的一处是什么？
```

然后根据你的回答继续追：

- 如果你还在复述作者，它会让你换成自己的话。
- 如果你讲了个人经历，它会问这个经历对应材料里的哪一个点。
- 如果你一直绕着一个例子转，它会把你带回材料的其他部分。
- 如果你答不上来，它会降低问题台阶，而不是替你讲答案。
- 如果访谈变长，它会维护一份本地记录，避免后面忘掉前面讲过什么。

它的目标不是让 AI 显得很懂。

它的目标是让你听见自己到底懂到哪里。

## 结束后能得到什么

访谈结束时，它会先给你一个暂停点：这一轮主要聊清了什么，哪些材料部分还没展开，下次可以从哪里继续。

如果素材足够，它会问你要不要导出一篇小文章。

这篇文章不是材料摘要，也不是 AI 替你写观点。它会以你刚才说出的理解为原料，保留你的口吻、节奏和关键表达，再做必要的组织和清理。

它可以作为：

- 一篇文章的初稿。
- 一段口播分享的底稿。
- 一次读书会分享的结构。
- 一份给自己看的深度复盘。

如果素材不够，它不会硬写，也不会为了凑文章重新补问。

## 安装

先把这个仓库安装到你的 AI 智能体的 skills 目录里。

如果你的智能体支持从 GitHub 安装或加载 skill，可以直接跟 Codex、Claude Code 或其他 AI 编程智能体说：

```text
请从 https://github.com/LuKK351/lukk-echo-interviewer 安装这个 skill。
```

安装完成后，重启你的 AI 客户端或智能体环境，让它重新加载 skills。

其他支持本地 skills 的智能体，也可以把整个仓库作为一个 skill 目录加载；核心入口是 `SKILL.md`，参考文件在 `references/`。

## 使用

安装完成后，你可以直接把材料发给 AI，然后用自然语言说明你想做一场“材料内化访谈”。

如果你的环境支持点名 skill，直接写 `$lukk-echo-interviewer` 会更稳：

```text
请使用 $lukk-echo-interviewer 围绕这份材料采访我，帮我看看哪些内容已经变成了我自己的理解。
```

也可以不写 skill 名称，用更自然的说法触发：

```text
我刚读完这篇文章，你来采访我，看看我是不是真的理解了。
```

```text
围绕这本书问我问题，不要总结，帮我把理解讲出来。
```

```text
我想把这期播客的内容讲给别人听，但不知道从哪里开始。你先采访我。
```

你可以提供原文、文件路径、URL、转录稿、笔记或摘录。

如果只给书名、文章名或节目名，而当前环境拿不到原文，它会请你补充材料内容。

## 本地记录

长访谈默认会保存一份轻量记录，用来防止上下文丢失和访谈跑偏。

默认目录类似：

```text
./lukk-echo-sessions/YYYY-MM-DD-HHMM-<material-slug>/
```

可能生成：

- `interview.md`：访谈记录和恢复点。
- `article.md`：只有你确认导出文章后才会生成。

如果你不想保存，开场时直接说关闭保存即可。

## 一句话

`lukk-echo-interviewer` 不是帮你多学一点。

它是帮你确认：你学过的东西，能不能从你嘴里长出来。
